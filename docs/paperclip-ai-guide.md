# Paperclip — AI 에이전트를 직원처럼 관리하는 오픈소스 오케스트레이터

AI 에이전트 하나로는 부족한 시대. 여러 에이전트를 팀으로 구성하고, 조직도를 만들고, 예산을 관리하고, 업무를 자동으로 배분하는 도구가 나왔습니다. Paperclip입니다.

---

## Paperclip이란?

Paperclip은 **여러 AI 에이전트를 하나의 조직으로 관리**하는 오픈소스 오케스트레이션 플랫폼입니다. GitHub 30,700+ stars, 2026년 3월 기준 가장 주목받는 에이전트 관리 도구입니다.

쉽게 말하면, AI 에이전트를 "직원"처럼 채용하고, 조직도에 배치하고, 업무를 할당하고, 비용을 추적하는 시스템입니다.

겉보기에는 일반적인 태스크 매니저처럼 보이지만, 내부에는 조직도, 예산 관리, 거버넌스, 목표 정렬, 에이전트 간 협업 구조가 갖춰져 있습니다.

---

## 왜 필요한가?

AI 에이전트를 하나만 쓸 때는 문제가 없습니다. 터미널에서 Claude Code를 실행하고, 결과를 확인하면 됩니다.

하지만 에이전트가 3개, 5개, 10개로 늘어나면?

- 누가 무슨 작업을 하고 있지?
- 토큰 비용이 얼마나 나가고 있지?
- 에이전트 A의 작업이 끝나야 에이전트 B가 시작할 수 있는데?
- 에이전트가 잘못된 방향으로 갈 때 어떻게 멈추지?

이런 문제를 해결하는 게 Paperclip입니다.

## 핵심 기능

### 1. 조직도 (Org Chart)

에이전트에게 역할과 보고 체계를 부여합니다. 마치 회사 조직도처럼요.

```
CEO Agent
├── CTO Agent
│   ├── Frontend Dev Agent
│   ├── Backend Dev Agent
│   └── QA Agent
├── Design Lead Agent
└── Content Agent
```

각 에이전트는 상위 에이전트의 지시를 받고, 하위 에이전트에게 업무를 위임할 수 있습니다.

### 2. 목표 계층 (Goal Alignment)

모든 태스크에는 "왜 이 작업을 하는가"가 연결되어 있습니다.

```
회사 목표: 월 매출 1000만원
  └── 팀 목표: 블로그 트래픽 2배 증가
       └── 에이전트 태스크: SEO 최적화 포스트 10개 작성
            └── 서브태스크: 키워드 리서치
```

에이전트는 자신이 "왜" 이 작업을 하는지 항상 알고 있습니다. 단순히 다음 단계만 아는 게 아닙니다.

### 3. 예산 관리 (Budget Control)

모든 에이전트에게 토큰 예산이 있습니다.

- 에이전트별 일일/월간 토큰 한도
- 모든 도구 호출에 비용이 책정됨
- 예산 초과 시 자동 중단
- 비용 대시보드에서 실시간 추적

예산 없이 에이전트를 돌리면 토큰 비용이 눈덩이처럼 불어납니다. Paperclip은 이걸 원천적으로 막아줍니다.

### 4. 감사 추적 (Audit Trail)

모든 에이전트의 모든 행동이 기록됩니다.

- 어떤 도구를 호출했는지
- 어떤 API를 요청했는지
- 어떤 결정을 내렸는지
- 언제, 왜

기록은 추가 전용(append-only)이라서 편집이나 삭제가 불가능합니다. 문제가 생기면 정확히 무슨 일이 일어났는지 추적할 수 있습니다.

### 5. 승인 게이트 (Approval Gates)

위험한 작업 전에 승인을 요구할 수 있습니다.

- 프로덕션 배포 전 승인
- 외부 API 호출 전 승인
- 비용이 일정 금액 초과 시 승인

설정 변경은 버전 관리되어 잘못된 변경을 롤백할 수 있습니다.

### 6. 에이전트 호환성

Paperclip은 특정 에이전트에 종속되지 않습니다.

지원하는 에이전트:
- Claude Code
- OpenAI Codex
- OpenClaw
- Gemini CLI
- Cursor
- Hermes
- Bash 스크립트
- HTTP Webhook (어떤 에이전트든 연결 가능)

"당신의 에이전트를 가져오세요. Paperclip이 조율합니다."

## 아키텍처

```
┌─────────────────────────────────────┐
│         Paperclip Dashboard          │
│    (React UI — 조직도, 태스크, 예산)   │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│       Paperclip Server (Node.js)     │
│  ┌──────────┐  ┌──────────────────┐ │
│  │ Task     │  │ Budget           │ │
│  │ Manager  │  │ Controller       │ │
│  └──────────┘  └──────────────────┘ │
│  ┌──────────┐  ┌──────────────────┐ │
│  │ Org      │  │ Audit            │ │
│  │ Chart    │  │ Logger           │ │
│  └──────────┘  └──────────────────┘ │
└──────────────┬──────────────────────┘
               │
        ┌──────┼──────┐
        ▼      ▼      ▼
   Claude   Codex  OpenClaw  ...
   Code     Agent   Agent
```

- **서버**: Node.js 단일 프로세스
- **DB**: 로컬은 내장 Postgres, 프로덕션은 외부 Postgres 연결
- **UI**: React 대시보드
- **에이전트 연결**: 어댑터 패턴 (플러그인)

## 시작하기

### 설치 (30초)

```bash
npx paperclipai onboard --yes
```

이 명령 하나로:
1. 내장 PostgreSQL 자동 생성
2. 첫 번째 "회사" 설정
3. API 서버 시작 (http://localhost:3100)

별도 데이터베이스 설치가 필요 없습니다.

### 에이전트 등록

```bash
npx paperclipai agent add \
  --name "SecurityAuditor" \
  --type "claude-code" \
  --budget-daily 100000
```

### 태스크 생성

대시보드에서 태스크를 만들고 에이전트에게 할당하면, Paperclip이 스케줄에 따라 에이전트를 실행합니다.

---

## 실행 모델

Paperclip의 에이전트 실행 방식은 두 가지입니다.

### 1. 하트비트 기반 (기본)

에이전트가 일정 주기로 "깨어나서" 할 일을 확인합니다.

```
[스케줄] → 에이전트 깨어남 → 태스크 확인 → 작업 → 결과 보고 → 잠듦
```

### 2. 이벤트 기반

특정 이벤트가 발생하면 에이전트가 트리거됩니다.

```
[태스크 할당] → 에이전트 트리거 → 작업 → 결과 보고
[@멘션] → 에이전트 트리거 → 응답
```

태스크 체크아웃과 예산 차감은 원자적(atomic)으로 처리됩니다. 두 에이전트가 같은 태스크를 중복 실행하거나, 예산 초과 상태에서 작업이 시작되는 일이 없습니다.

## 어떤 경우에 적합한가?

### Paperclip이 맞는 경우

- 5개 이상의 에이전트를 동시에 운영
- 에이전트 간 협업이 필요 (A의 결과를 B가 사용)
- 토큰 비용 관리가 중요
- 감사 추적이 필요 (규정 준수, 보안)
- 여러 종류의 에이전트를 혼합 사용 (Claude + Codex + 커스텀)

### Paperclip이 과한 경우

- 에이전트 1-2개만 사용
- 단순한 자동화 (cron + 스크립트로 충분)
- 빠른 프로토타이핑

조직도, 목표 계층, 승인 워크플로우를 설정하는 오버헤드가 있기 때문에, 소규모에서는 배보다 배꼽이 클 수 있습니다.

## 비슷한 도구들

| 도구 | 특징 | 차이점 |
|------|------|--------|
| **Paperclip** | 에이전트 조직 관리 | 조직도, 예산, 거버넌스 |
| **OpenAI Symphony** | Linear 칸반 + 에이전트 | Linear 보드 기반, OpenAI 전용 |
| **CrewAI** | 에이전트 역할 정의 | Python 코드 기반, UI 없음 |
| **AutoGen** | 멀티 에이전트 대화 | 연구 목적, 프로덕션 미흡 |
| **LangGraph** | 에이전트 워크플로우 | 그래프 기반, 저수준 |

Paperclip의 차별점은 "회사처럼 운영"한다는 점입니다. 다른 도구들이 "에이전트를 실행"하는 데 초점을 맞추는 반면, Paperclip은 "에이전트를 관리"하는 데 초점을 맞춥니다.

## 주의사항

### 비용

Paperclip 자체는 무료(오픈소스)이지만, 에이전트가 사용하는 AI API 비용은 별도입니다. 에이전트 5개가 동시에 돌면 토큰 비용이 급격히 증가합니다. 반드시 예산 한도를 설정하세요.

### 복잡성

"직원을 채용"하는 것처럼 에이전트를 설정해야 합니다. 역할 정의, 조직도 구성, 목표 설정, 예산 배분... 이 초기 설정 비용이 있습니다. 에이전트 2-3개라면 터미널에서 직접 실행하는 게 더 빠를 수 있습니다.

### 에이전트 품질

Paperclip은 "관리 도구"이지 "에이전트"가 아닙니다. 에이전트 자체의 품질은 사용하는 AI 모델에 달려 있습니다. Paperclip이 아무리 잘 관리해도, 에이전트가 잘못된 코드를 작성하면 잘못된 코드가 나옵니다.

### 보안

에이전트에게 프로덕션 환경 접근 권한을 줄 때는 신중해야 합니다. 승인 게이트를 적극 활용하고, 최소 권한 원칙을 적용하세요.

## 마치며

AI 에이전트가 1명일 때는 직접 지시하면 됩니다.
5명이 되면 관리가 필요합니다.
10명이 되면 조직이 필요합니다.

Paperclip은 그 "조직"을 만들어주는 도구입니다.

아직 초기 단계이고, 2026년 3월에만 3번의 메이저 업데이트가 있었을 만큼 빠르게 발전하고 있습니다. 에이전트를 활용한 자동화를 고민하고 있다면, 한번 설치해보고 느껴보세요.

```bash
npx paperclipai onboard --yes
```

이 한 줄이면 시작할 수 있습니다.
